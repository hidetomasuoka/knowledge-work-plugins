# 環境設定・デプロイ情報

## 環境一覧

| 環境 | 用途 | URL |
|-----|-----|-----|
| local | 開発 | http://localhost:3000 |
| staging | テスト・検証 | https://staging.example.com |
| production | 本番 | https://www.example.com |

## デプロイ順序

<!-- サービス間の依存がある場合はデプロイ順序を明記 -->

```
1. infra（Terraform apply）
2. backend
3. frontend
```

## 本番 / STG の主な差分

<!-- 環境差異が原因でバグが起きやすい設定を記載 -->

| 設定 | STG | 本番 | 注意点 |
|-----|-----|-----|-------|
| メール送信 | モックを使用 | 実送信 | STG では実際にメールが飛ばない |
| 決済 | Stripe テストモード | 本番モード | カード番号が異なる |
| ログレベル | DEBUG | INFO | STG のみ詳細ログが出る |

## ロールバック手順

<!-- 問題発生時の手順概要 -->

```
1. 問題を確認したら Slack #incidents に報告
2. 前バージョンのタグで再デプロイ
3. DB マイグレーションがある場合は別途ロールバック手順を確認
```

---

## カスタマイズ方法

1. 環境一覧に自社の環境を追記（本番/STG/開発 など）
2. デプロイ順序を実際の手順に書き換える
3. 本番/STG の差分として、バグの原因になりやすい設定を記録する
4. ロールバック手順を実際のフローに書き換える
