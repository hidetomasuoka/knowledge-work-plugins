# Company Intel

企業URLを渡すと、どんな情報が欲しいかをインタラクティブに確認してから調査・まとめます。必要な情報だけを、必要な深さで取得できます。

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│                      COMPANY INTEL                               │
├─────────────────────────────────────────────────────────────────┤
│  STEP 1: URLを受け取る                                            │
│  STEP 2: 欲しい情報をインタラクティブに選択                         │
│  STEP 3: 選択した項目だけをウェブ検索で調査                         │
│  STEP 4: 選択項目に絞った構造化レポートを出力                       │
└─────────────────────────────────────────────────────────────────┘
```

---

## Getting Started

URLを渡すだけでOKです:

- 「https://example.com について調べて」
- 「この会社の情報をまとめて: https://example.com」
- 「https://example.com の企業リサーチをして」

URLがまだない場合は聞きます。

---

## 選択できる情報カテゴリ

| # | カテゴリ | 内容 |
|---|----------|------|
| 1 | **会社概要・事業内容** | 何をしている会社か、業界、規模、設立年、本社所在地 |
| 2 | **経営陣・キーパーソン** | CEO・役員・創業者の経歴、意思決定者の情報 |
| 3 | **財務・資金調達** | 売上推定、資金調達履歴、投資家、上場有無 |
| 4 | **最新ニュース・動向** | 直近のプレスリリース、メディア露出、発表事項 |
| 5 | **採用・組織動向** | 求人情報、採用トレンド、組織の成長シグナル |
| 6 | **製品・サービス詳細** | 提供しているプロダクト・サービスの詳細、価格帯、対象顧客 |
| 7 | **競合・市場ポジション** | 主要競合、差別化ポイント、市場でのポジション |
| 8 | **技術スタック** | 使用している技術、ツール、インフラ（推定） |

## Execution Flow

### Step 1: URL確認

```
ユーザーからURLを受け取る。
URLがない場合:
  「調査したい企業のURLを教えてください。」
```

### Step 2: インタラクティブな情報選択

URLを受け取ったら、必ず以下の形式で確認する:

```
「[企業名 or ドメイン] について調査します。

どの情報が必要ですか？番号で選んでください（複数選択可、「全部」でも可）:

1. 会社概要・事業内容
2. 経営陣・キーパーソン
3. 財務・資金調達
4. 最新ニュース・動向
5. 採用・組織動向
6. 製品・サービス詳細
7. 競合・市場ポジション
8. 技術スタック

例: 「1, 2, 4」 または 「全部」」
```

### Step 3: 選択に基づいて検索実行

ユーザーの選択に応じて、以下の検索を実行する:

**1. 会社概要・事業内容 が選択された場合:**
```
検索: "[企業名]" "[企業名] about" "[企業名] 事業内容"
抽出: 事業説明、業界、従業員数、設立年、本社
```

**2. 経営陣・キーパーソン が選択された場合:**
```
検索: "[企業名] CEO" "[企業名] 代表取締役" "[企業名] leadership team" "[企業名] founders"
抽出: 氏名、役職、経歴、LinkedInプロフィール
```

**3. 財務・資金調達 が選択された場合:**
```
検索: "[企業名] funding" "[企業名] 資金調達" "[企業名] revenue" "[企業名] investors" "[企業名] IPO"
抽出: 調達額・ラウンド・投資家、売上推定、上場状況
```

**4. 最新ニュース・動向 が選択された場合:**
```
検索: "[企業名] news" "[企業名] press release" "[企業名] 2024 2025" "[企業名] announcement"
抽出: 直近90日のニュース・発表事項（日付付き）
```

**5. 採用・組織動向 が選択された場合:**
```
検索: "[企業名] careers" "[企業名] jobs" "[企業名] hiring" "[企業名] 採用"
抽出: 求人数、採用部門、採用トレンド
```

**6. 製品・サービス詳細 が選択された場合:**
```
検索: "[企業名] product" "[企業名] service" "[企業名] pricing" "[企業名] customers"
抽出: プロダクト名・機能・価格帯・対象顧客
```

**7. 競合・市場ポジション が選択された場合:**
```
検索: "[企業名] competitors" "[企業名] vs" "[企業名] market share" "[企業名] 競合"
抽出: 主要競合、強み・弱み、市場ポジション
```

**8. 技術スタック が選択された場合:**
```
検索: "[企業名] tech stack" "[企業名] technology" "[企業名] engineering blog" site:stackshare.io "[企業名]"
抽出: 使用技術・フレームワーク・インフラ
```

### Step 4: レポート生成

選択された項目のみセクションとして出力する。選択されていない項目は省略する。

## 重要なルール

1. **必ずStep 2のインタラクティブな確認を行う** — URLを受け取っても、情報カテゴリを確認する前に調査を始めない
2. **選択されたカテゴリのみ調査・出力する** — 選択されていない項目は一切出力しない
3. **「全部」と言われた場合はすべてのカテゴリを調査する**
4. **情報が見つからない場合は「情報が見つかりませんでした」と明記** — 推測で埋めない
5. **調査後に追加調査を提案する** — 「他に知りたいことがあれば追加で調べます」
